7. Что такое API?

API (Application programming interface) - программный 
интерфейс приложения. API - это составляющая часть сервера, которая
получает запросы 
и посылает ответы. По сути, это описания взаимодействия приложений между 
собой, обмен данными. Например: вы заходите в приложение для покупки 
билетов, а это приложение показывает погоду того города, куда вы 
намереваетесь лететь. Сам сервис (сайт, приложение, сервис) не имеет 
метеорологических датчиков по всему миру, оно просто взаимодействует еще с 
одним приложением (сайтом), которое выводит информацию в искомое 
приложение. А ещё любая оплата (покупка билетов или любого товара онлайн) 
идет через API,когда мы нажимаем на "оплати", сервис "перебрасывает" нас 
на платежную систему. Другими словами, это договор взаимодействия между 
программами, сайтами и сервисами. При помощи API и его контрактов можно 
регистрироваться на различных сервисах через Google, Facebook, VK и другие 
сервисы.

Зачем это тестировщику?

Существует тестирование API, а точнее двух программ находящихся на разных 
компьютерах или серверах и как-то взаимодействующих друг с другом. 
Тестировщик проверяет функциональность и способность работать согласно 
ожиданиям бизнес-логики. Так тестировщик может автоматизировть процесс 
тестирования, особенно это актуально при заполнении данных при авторизации 
на сайтах через другие сервисы.

9. URL, (URI, URL, URN):

URI - uniform resuorce identifier - это строка символов или знаков, 
которая идентифицирует ресурс по его адресу или по его названия (или по 
адресу и названию вместе).

URL - uniform resource locator - это строка символов, которая 
идентифицирует ресурс только по его АДРЕСУ.

URN - uniform resource name - это строка символов, которая идентифицирует 
ресурс только по его имени.

Таким образом, URL и URN являются подвидами URI, содержащими место 
хранения или уникальное имя в определенном пространстве. Например, книга 
может содержать уникальное имя "0-395-36341-1 в пространстве имён ISBN, 
но,в отличие от URL,URN не указывает на местонахождение этого ресурса: в нём 
не сказано, в каком магазине её можно купить или на каком сайте скачать. 
Впрочем, в последнее время появилась тенденция говорить просто URI о любой 
строке-идентификаторе, без дальнейших уточнений. Так что, возможно, 
термины URL и URN скоро уйдут в прошлое"[https://ru.wikipedia.org/wiki/URI].

10. Идемпотентные HTTP методы - идемпотентными методами называтся запросы 
на сервер, отправленные за n-количество времени n-количество раз, но 
которые не изменяют состояния сервера от первого запроса. Данные запросы 
могут учитываться в статистике. Так, идемпотентными запросами могут быть 
GET, HEAD, PUT, LELETE, но не POST. Идемпотентные методы необходимы, чтобы 
несколько раз отправленный запрос не был интерпритирован, как разные 
запросы. Например, неский Василий, отправляет в телефонном приложении 
запрос для такси. В этот момент, у него происходит обрыв мобильной связи и 
ответ об успешной отправки он не получает. Он посторяет второй раз попытку 
и в итоге к нему приезжает две машины, которые он оплачивает. Для 
предотвращения таких ситуаций пишут идемпотентный код.

12. Иденфикация, Аутентификация, Авторизация

Идентификация - это отождествления объекта на неизвестным объектом на 
основании определенных признаков. То есть, в качестве примера в WEB, можно 
рассмотреть следующее: человек зашел на сай социальной сети и указал, что 
он Иванов Иван. Также, можно дать следующее определение: идентификация - 
это процес распознания субъекта по его идентификатору (уникальный признак 
объекта, который может отличить его от других).

Аутентификация - это проверка подлинности личности. Существуют различные 
виды аутентификации: через многоразовый или одноразовый пароль, смс, 
отпечаток пальца, face-id и т.д.

Авторизация - это предоставления определенному лицу или группе лиц на 
выполнение определенных дейсвий.

Таким образом, идентификация, аутентификация и авторизация, это разные по 
своему значению и действию понятия. Для того, чтобы войти на личную 
страничку в социальной сети, сперва проходит процесс идентификации, где 
происходит определение личности, после чего личность подтверждает право на 
распоряжение и владение этим ресурсом - это процес аутентификации, и 
только потом происходит авторизация.

13. Что такое IP (internet protocol) - это уникальный адрес устройства или 
сервера. Выгляди он как набор цифр от 0 до 255 и разделен точками. 
например: 255.255.30.04.  IP необходим для того, чтобы пользователь мог 
послать запрос и получить ответ. IP может быть динамическим и статическим. 
Статический не изменяется никогда, например адрес сайта Goole, 
динамический, соответственно, изменяется. IP присваивается любому 
устройству, который связан локальной сетью. Также его можно 
дифференцировать на внутренний и внешний. Внешний выдает провайдер, а 
внутренний роутер для ваших устройств. Гаджеты могу иметь один IP-адрес, 
но отличаться портами (разделяется двоеточиями (:) 246.263.98.2:5005 - 
последние цифры после двоеточия - это и есть так называемый порт).

14. Что такое октеты в DNS
DNS - domain name system - система доменных имен. DNS - это компьютерная 
распределенная система о получении IP-адресов различных хостов.

"База имен является распределенной, так как нет такой ЭВМ, где бы 
хранилась вся эта информация. Как уже отмечалось имя содержит несколько полей 
(длиной не более 63 символов), разделенных точками. Имя может содержать не 
более 255 октетов, включая байт длины. Анализ имени производится справа 
налево. Самая правая секция имени характеризует страну (двухсимвольные 
национальные коды смотри в приложении), или характер организации 
образовательная, коммерческая, правительственная и 
т.д.)."[http://book.itep.ru/4/44/dns_4412.htm]. В информатике октет равен 
одному байту или 8 битам. Другими словали, адрес домена состоит из 
трехзначных чисел разделенных точками с четырехкратным повторением. 
Напрмер: 127.234.78.5 - акждое число разделенное точками и является 
октетом. Это запись в десятиричной системе. Тот же адрес, а точнее первый 
октет в двоичной системе выглядил бы примерно так - 10000000.

15. Что такое порт, сколько портов у Linux сервера
Порт - это число, указываемое при запросе на соединение. При раболе 
клиент-серверной архитектуре, каждая команда выполняется определенным 
образом, и чтобы регламинтировать и унифицировать эту работу, существует 
порт, то есть число посылаемое при запросе. Например, для просмотра 
web-страницы используют 80 порт. Максимальное количество портов у Linux 
сервера - 65536, то есть такое количество типов соединений может 
поддерживать данный сервер.

16. Уровни OSI (The open system interconnection model) - сетевая модель 
протоколов, посредствам который различные устройства в сети могут 
взаимодействовать. Выделяют 7 уровней передачи данных. С самого верхнего 
данная структура выгляди следующим образом:

1) Прикладной уровень (application) - верхний уровень, коорый позволяет 
взаимодействовать пользовательсим приложениям с сетью.

2) Уровень представления (presentation) - уровень шифрования и 
дешифрования. При передачи данных часто используются разные протоколы 
связи и шифрования. Именно для этого и сущетвует данный уровень.

3) Сеансовый уровень (session) - позволяет пользователям долгое время 
поддерживать соединения, синхронизировать данные.

4) Транспортный уровень (transport) - позволяет передавать информацию. 
Существует множество протоколов передачи данных, от простых и не 
защищенных до протоколов, которые обеспечивают шифрование и подтверждение 
доставки информации.

5) Сетевой уровень (network) - определяют канал передачи информации, то 
есть данный уровень является маршрутизатором между отправителем и 
получателем информации.

6) Канальный уровень (data link) - представляет взаимодействие сетей на 
физическом уровне, контрольошибок передачи данных и их исправление.

7) Физический уровень (Physical) - самый нижний уровень, определяет способ 
передачи данным (двоичной системы) (оптоволокно, спутниковый канал 
передачи данных и т.д.).
