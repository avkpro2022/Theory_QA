7. Что такое API?

API (Application programming interface) - программный 
интерфейс приложения. API - это составляющая часть сервера, которая
получает запросы 
и посылает ответы. По сути, это описания взаимодействия приложений между 
собой, обмен данными. Например: вы заходите в приложение для покупки 
билетов, а это приложение показывает погоду того города, куда вы 
намереваетесь лететь. Сам сервис (сайт, приложение, сервис) не имеет 
метеорологических датчиков по всему миру, оно просто взаимодействует еще с 
одним приложением (сайтом), которое выводит информацию в искомое 
приложение. А ещё любая оплата (покупка билетов или любого товара онлайн) 
идет через API,когда мы нажимаем на "оплати", сервис "перебрасывает" нас 
на платежную систему. Другими словами, это договор взаимодействия между 
программами, сайтами и сервисами. При помощи API и его контрактов можно 
регистрироваться на различных сервисах через Google, Facebook, VK и другие 
сервисы.

Зачем это тестировщику?

Существует тестирование API, а точнее двух программ находящихся на разных 
компьютерах или серверах и как-то взаимодействующих друг с другом. 
Тестировщик проверяет функциональность и способность работать согласно 
ожиданиям бизнес-логики. Так тестировщик может автоматизировть процесс 
тестирования, особенно это актуально при заполнении данных при авторизации 
на сайтах через другие сервисы.

9. URL, (URI, URL, URN):

URI - uniform resuorce identifier - это строка символов или знаков, 
которая идентифицирует ресурс по его адресу или по его названия (или по 
адресу и названию вместе).

URL - uniform resource locator - это строка символов, которая 
идентифицирует ресурс только по его АДРЕСУ.

URN - uniform resource name - это строка символов, которая идентифицирует 
ресурс только по его имени.

Таким образом, URL и URN являются подвидами URI, содержащими место 
хранения или уникальное имя в определенном пространстве. Например, книга 
может содержать уникальное имя "0-395-36341-1 в пространстве имён ISBN, 
но,в отличие от URL,URN не указывает на местонахождение этого ресурса: в нём 
не сказано, в каком магазине её можно купить или на каком сайте скачать. 
Впрочем, в последнее время появилась тенденция говорить просто URI о любой 
строке-идентификаторе, без дальнейших уточнений. Так что, возможно, 
термины URL и URN скоро уйдут в прошлое"[https://ru.wikipedia.org/wiki/URI].

10. Идемпотентные HTTP методы - идемпотентными методами называтся запросы 
на сервер, отправленные за n-количество времени n-количество раз, но 
которые не изменяют состояния сервера от первого запроса. Данные запросы 
могут учитываться в статистике. Так, идемпотентными запросами могут быть 
GET, HEAD, PUT, LELETE, но не POST. Идемпотентные методы необходимы, чтобы 
несколько раз отправленный запрос не был интерпритирован, как разные 
запросы. Например, неский Василий, отправляет в телефонном приложении 
запрос для такси. В этот момент, у него происходит обрыв мобильной связи и 
ответ об успешной отправки он не получает. Он посторяет второй раз попытку 
и в итоге к нему приезжает две машины, которые он оплачивает. Для 
предотвращения таких ситуаций пишут идемпотентный код.
